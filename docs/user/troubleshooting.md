# Troubleshooting — both platforms

What to check when Docent refuses, halts, or drops something. This document
is a by-symptom index: each entry names a symptom as the platform surfaces it
and routes to the document that owns the behaviour — the user guides
([extension](extension.md), [desktop](desktop-windows.md)) and the format,
protocol, and architecture owners each row names. Each row summarises the
owning rule so the symptom is recognisable; the owning document holds the
authoritative statement, and when a rule changes there, the matching row
here changes with it (this index's homing in the platform, sync, format,
and user areas keeps it in a change's disposition view).

## Import and export

Both platforms run the same shared import acceptance
([Shared Core](../architecture/system/shared-core.md#the-parity-rule)); the
[extension guide's Import section](extension.md#import) documents the checks.

| Symptom                                                      | What happened                                                                                                                                                                    | Where the rules live                                                                                                        |
| ------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| "Could not read file — make sure it is a valid .docent.json" | The file did not parse as JSON.                                                                                                                                                  | [Extension guide — Import](extension.md#import)                                                                             |
| "Import failed: file does not match the Docent format."      | The file failed the ingestion bounds (at most 10 MiB of JSON, nested at most 64 levels) or schema validation, including the `docent_format` version stamp. Nothing was imported. | [Session Format — Import acceptance](../technical/session-format.md#import-acceptance)                                      |
| A file that used to import is rejected after updating Docent | The version-stamp gate: a schema version bump stops previously exported files importing by design. Pin the producing version, or re-export from the current one.                 | [Session Format — Schema version pinning](../technical/session-format.md#schema-version-pinning)                            |
| Importing a project you already have produced a second one   | By design: an import never overwrites or merges — the copy gets a new identity and a " (copy)" name suffix.                                                                      | [Extension guide — Import](extension.md#import)                                                                             |
| Desktop: the import or export dialog was cancelled           | Not an error: cancelling the native open dialog imports nothing; cancelling the native save dialog writes nothing.                                                               | [Application shell — command surface](../architecture/application/desktop/windows/application-shell.md#the-command-surface) |
| Extension: looking for a file dialog                         | The extension uses the browser's flows: **Export** downloads the file, **Import** opens a file picker on the Projects list.                                                      | [Extension guide — Export](extension.md#export) / [Import](extension.md#import)                                             |

## Send (dispatch)

| Symptom                                                   | What happened                                                                                                                                                                                        | Where the rules live                                                                                                                       |
| --------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| **Send** is disabled                                      | No endpoint is configured, or the open project has no recording with at least one active step.                                                                                                       | [Extension guide — Send](extension.md#send); [Dispatch — send eligibility](../api/dispatch.md#send-eligibility-and-the-post-send-cooldown) |
| The endpoint URL is refused when saving Settings          | The save-time URL policy rejected it: the scheme must be `http://`/`https://`, no embedded credentials, no link-local IPv4 address, and `https://` is required alongside an API key except loopback. | [Dispatch — endpoint URL policy](../api/dispatch.md#endpoint-url-policy)                                                                   |
| **Send** is disabled right after a send, with a countdown | The 5-second post-send cooldown — a rapid-resend guard. A failed send starts no cooldown.                                                                                                            | [Dispatch — post-send cooldown](../api/dispatch.md#send-eligibility-and-the-post-send-cooldown)                                            |
| A send fails after several attempts                       | Transient failures (network error, timeout, `429`, `5xx`) are retried up to three times; when retries are exhausted the last error is surfaced. Any other status fails immediately and is shown.     | [Dispatch — retries](../api/dispatch.md#retries-and-duplicate-delivery) / [response handling](../api/dispatch.md#response-handling)        |
| A send is rejected before anything is sent (size)         | The payload exceeded the 50 MB outbound bound — send recordings individually.                                                                                                                        | [Dispatch — size bounds](../api/dispatch.md#size-bounds)                                                                                   |

## Sync

The sync controls and messages are documented once, in the
[extension guide's Sync section](extension.md#sync), which the
[desktop guide](desktop-windows.md#sync) adopts.

| Symptom                                      | What happened                                                                                                                                                                                                   | Where the rules live                                                                  |
| -------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| A sync halted with a message                 | The four halt messages — recording in progress, uncommitted actions, the protective no-changes halt, and the authentication halt — each say what to do next.                                                    | [Extension guide — What a sync does](extension.md#what-a-sync-does)                   |
| Auto-sync turned itself off                  | An authentication failure during any sync disables Auto-sync. Fix the API key, then test the connection again to re-enable it.                                                                                  | [Extension guide — Auto-sync](extension.md#auto-sync)                                 |
| The Auto-sync toggle is disabled             | It needs a configured sync server and a connection test passed against the current settings; saving changed settings invalidates the previous test.                                                             | [Extension guide — Configure the sync server](extension.md#configure-the-sync-server) |
| The connection test failed                   | The status line distinguishes rejected credentials from an unreachable server, each with its remedy.                                                                                                            | [Extension guide — Test the connection](extension.md#test-the-connection)             |
| Rows carry **Review** or **Conflict** badges | The server changed something you have not touched (review), or the same item changed on both sides (conflict). A rolled-up badge on a project row opens the project; any other badge opens its resolution view. | [Extension guide — Attention badges](extension.md#attention-badges)                   |
| The sync summary reports skipped projects    | Projects whose format stamp does not match your platform or schema version are skipped and reported with the reason — never merged.                                                                             | [Extension guide — What a sync does](extension.md#what-a-sync-does)                   |

## Capture

| Symptom                                               | What happened                                                                                                                                                                                           | Where the rules live                                                                                                                                                                                           |
| ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Extension: capture paused with a storage warning      | Storage pressure: capture auto-pauses at 8 MiB of the 10 MiB quota, with an explicit keep-recording override. Actions performed while paused are not captured; space is freed only by your own actions. | [Persistence — storage pressure](../architecture/system/persistence.md#storage-pressure); [Extension capture — storage pressure](../architecture/application/extension/capture-principles.md#storage-pressure) |
| Desktop: uncommitted actions were gone after quitting | The desktop pending buffer lives only in the running app: quitting discards uncommitted captured actions; committed steps are already in the session file.                                              | [Desktop guide — Your data](desktop-windows.md#your-data); [Persistence](../architecture/system/persistence.md#deliberately-ephemeral-state)                                                                   |
| Extension: pending actions vanished when reopening    | Opening or creating any recording discards the pending list (a browser restart preserves it). The pending-list rules are tabled in the guide.                                                           | [Extension guide — Pending actions](extension.md#pending-actions)                                                                                                                                              |
