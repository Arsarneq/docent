Docent Manual Test — Browser Chrome Interactions

Instructions

These 7 tests involve browser chrome interactions that cannot be automated with Playwright (CDP cannot trigger browser-level shortcuts or interact with native widgets). All other capture tests are automated (48 tests).

Start recording in Docent, perform each test, commit a step after each.

Test 1: Bookmark Bar Click

Click a bookmark in the bookmark bar, then click the browser Back button to return.

Expected: 1 navigate(auto_bookmark) + 1 navigate(back_forward). Both are browser chrome actions captured via their immediate effects. No click events (bookmark bar and back button are browser chrome).

Test 2: Right-Click → Open Link in New Tab

Right-click the link below and choose "Open link in new tab" from the context menu.

Expected: 1 right_click on the link + 1 navigate(link) on the new tab. The right_click is the user's in-page action. The navigate(link) is the proxy for the context menu selection (a browser chrome action). No context_open (the new tab has an openerTabId so it's suppressed).

Right-click me → Open in new tab

Test 3: Ctrl+L (Focus Address Bar)

Press Ctrl+L to focus the address bar, then press Escape to cancel.

Expected: Nothing captured. Ctrl+L is intercepted by the browser before reaching the page. Escape in the address bar is also browser chrome.

Test 4: Arrow Keys in Native Select

Click the select to open it, press ↓ twice, press Enter to confirm.

Expected: 1 click (open dropdown) + 1 select (final selection).

Known limitation: Arrow key presses and Enter inside native select widgets are handled by the browser's internal widget layer before reaching the DOM. They are not capturable. The select action captures the user's final intent.

Test 5: Tab Key with Body Focused

Click on empty space (page background), then press Tab.

Expected: 1 key(Tab) with element body. The user pressed Tab — that's what they did, captured regardless of what was focused.

Test 6: Browser Back/Forward Button

Navigate somewhere (e.g. use Test 1), then click the browser's Back button.

Expected: 1 navigate(back_forward). The back button is browser chrome — the navigate is the immediate effect captured as proxy.

Test 7: Ctrl+T → Close Tab → Return

Press Ctrl+T to open a new tab, close it (Ctrl+W or click X), then click this tab to return.

Expected: 1 context_open (proxy for Ctrl+T) + 1 context_close (proxy for closing the tab) + 1 context_switch (proxy for clicking back to this tab).

Cannot be automated — Playwright's CDP dispatch does not trigger browser-level keyboard shortcuts.
