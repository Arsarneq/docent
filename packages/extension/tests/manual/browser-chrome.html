Docent Manual Test — Browser Chrome Interactions

Instructions

These 6 tests involve browser chrome (address bar, bookmarks, context menus) that cannot be automated with Playwright. All other capture tests are automated.

Start recording in Docent, perform each test, commit a step after each.

Test 1: Bookmark Bar Click

Click a bookmark in the bookmark bar (if visible), then navigate back.

Expected: 1 navigate action (nav_type: "auto_bookmark"). No click event (bookmark bar is browser chrome). No duplicate SPA navigate.

Test 2: Right-Click → Open Link in New Tab

Right-click the link below and choose "Open link in new tab" from the context menu.

Expected: 1 right_click on the link. The context menu selection itself is not capturable. The new tab opening (context_open + navigate) should NOT be captured — it's a side-effect of the menu selection.

Right-click me → Open in new tab

Test 3: Ctrl+L (Focus Address Bar)

Press Ctrl+L to focus the address bar, then press Escape to cancel.

Expected: Nothing captured. The address bar is browser chrome — the shortcut is intercepted before reaching the page.

Test 4: Arrow Keys in Native Select

Click the select below to open it, then press ↓ twice to navigate options, then press Enter to confirm.

Expected: 1 click + key(ArrowDown) events + 1 select action. Note: native select widgets may swallow keydown events before they reach the document.

Test 5: Tab Key with Body Focused

Click on empty space (page background) so nothing is focused, then press Tab.

Expected: No key(Tab) captured (activeElement is body, which is filtered). A focus event on whatever element receives focus IS expected (user navigating via keyboard). The browser may also scroll to the focused element.

Test 6: Browser Back/Forward Button

Navigate to another page (e.g. click Test 2's link), then click the browser's physical Back button (not a keyboard shortcut).

Expected: 1 navigate action (nav_type: "back_forward"). No click event (the back button is browser chrome).

Test 7: Ctrl+T (New Tab)

Press Ctrl+T to open a new tab, then close it and return to this tab.

Expected: Nothing captured. Ctrl+T is a browser shortcut (not capturable). The tab opening, switching, and closing are all side-effects of that shortcut. No context_open, context_switch, or context_close actions should appear.

Note: This test cannot be automated — Playwright's keyboard dispatch does not trigger browser-level shortcuts via CDP.
